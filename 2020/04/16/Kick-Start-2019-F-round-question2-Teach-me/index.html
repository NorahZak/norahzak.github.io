Kick Start 2019 F round question2: Teach me

2020-04-16

#kickstart #programming

问题描述

Here at Google we love teaching new skills to each other! There are N employees at Google, numbered from 1 to N. There are a total of S different skills, numbered from 1 to S. Each employee knows up to 5 different skills.

The i-th employee can mentor the j-th employee if there is a skill that the i-th employee knows that the j-th employee does not know. How many ordered pairs (i, j) are there where the i-th employee can mentor the j-th employee?

Limits:

1 ≤ T ≤ 100.
1 ≤ S ≤ 1000.
1 ≤ Ci ≤ 5 for all i.
1 ≤ Aij ≤ S for all i and j.
Aij ≠ Aik for all j ≠ k.

Test set 1 (Visible)

2 ≤ N ≤ 500.

Test set 2 (Hidden)

2 ≤ N ≤ 5 × 104.

解决方案

  1. 对每个(i, j)对，检查i能否教j，也就是i是否有j没有的技能。由于每个人最多掌握5种技能，因此这个算法的时间复杂度为O(5^2*N^2)
  2. 对员工i，假设能教他的人数为m(i)，那么最终的答案就是所有m(i)的和。而求m(i)，可以通过反过来求不能教员工i的人数，也就是那些掌握的技能是员工i技能的子集的人数。考虑到每个人最多只有5种技能，因此员工i技能集的子集B最多只有32个。为了求出技能集恰好等于B的员工人数，可以在预处理时，先将所有人以其掌握的技能集为键值插入哈希表中，这样就能在常数时间内求出恰好具有某一技能集B的人数。总的时间复杂度为O(2^5*N)
