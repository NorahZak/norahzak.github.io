1.std::size_type与int变量的比较

通常两者之间的比较可以正常进行。但是，当int型变量为一个负值时，判断会出现问题。比如：

int i = -1;
vector<int> a(2);
if (i < a.size())
		printf("Ok\n");

按照预想，-1<2，程序应该会输出”Ok”，然而并没有。

原因是，如果把负数与size_type类型的变量比较，不论这个比较关系式实际的值为真还是为假，最后都会返回false。

如果将程序改为下面所示，就能输出正确结果。

int i = -1;
vector<int> a(2);
if (i+1 < a.size()+1)
		printf("Ok\n");

2.优先队列：prioirity_queue的使用

prioirity_queue是C++中的一种数据结构，它相当于一个堆。

堆作为一种排序的数据结构，与线性的有序数组相比，它有如下优缺点：

优点：可在O(log(n))时间内插入或删除元素，而线性数组需要O(n)的时间

缺点：尽管可在O(n)时间内查询最大元素，但无法在O(n)时间内查询任意序数的元素，而线性数组可以做到这一点。或者说，堆是一种半排序的数据结构，而线性有序数组则是一种全序结构。

综合以上两点，堆的适用场景为：

1.集合中的数据频繁增删，需要大量插入和删除操作；

2.我们只关心集合中的最大元素，而不关心其他元素的相对顺序。

初始化：在prioirity_queue的模板列表中，三个参数的含义分别为：元素类型，元素的组织形式，比较函数。默认情况下，以vector作为元素的组织形式，排列顺序为大顶堆

struct cmp{
    bool operator()(int a, int b){
    		// 注意，如果你希望b在优先队列中的顺序排在a的前面
    		// 那么就令该函数返回true
    		// 例如，以下比较函数规定了一个大顶堆
    		return a < b;
    }
};
# 以下两条语句等效
priority_queue<int, vector<int>, cmp> mypq;
priority_queue<int> mypq;

插入元素：push()

mypq.push(4);
mypq.push(1);
mypq.push(7);

查询堆顶元素：top()

int a = mypq.top()    # a=7

删除堆顶元素：pop()

mypq.pop()

判别堆是否为空：empty()

mypq.empty()

3.pair的使用

pair是C++中的一种数据类型，它是由两个任意类型的元素组成的一个二元组。

初始化：pair<type1, type2> name = {value1, value2};

pair<int, string> mypair = {1, "one"};    
vector<pair<int, string>> = {mypair};

访问元素：first, second

int a = mypair.first;    # a=1
string s = mypair.second;    # s="one"
