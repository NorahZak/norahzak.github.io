4.笔记

操作系统

  1. 系统加速比=加速前时间/加速后时间=加速后速度/加速前速度
  2. 常用进程调度算法：先来先服务/时间片轮转/优先级调度/多级队列调度/短作业优先
  3. 加密算法：按照加密的秘钥与解密的秘钥是否相同，可以分为对称和非对称两类
    • 对称：DES、TripleDES、RC2、RC4、RC5、Blowfish、AES
    • 非对称（公钥加密，私钥解密）：RSA、Elgamal、背包算法、Rabin、D-H、ECC
  4. 安全性认证算法：
    • DSA：私钥签名，公钥验证
    • MD5：利用原文档的散列值来判断文档是否正确和完整
    • SHA-2
  5. 多线程环境下，进行读操作之前需要先加锁，以防止读的过程中数据被修改。

数据结构

  1. 具有特殊性质的图：

    • 哈密顿图：通过图G的每个结点一次，且仅一次的通路（回路），就是哈密顿通路（回路）。存在哈密顿回路的图就是哈密顿图。
    • 欧拉图：欧拉图是指通过图（无向图或有向图）中所有边且每边仅通过一次通路，相应的回路称为欧拉回路。具有欧拉回路的图称为欧拉图。

  2. 堆排序：

    • 自底向上建堆，时间复杂度O(n)，证明
    • 自顶向下调整堆，时间复杂度O(log(n)) * n.

计算机组成原理

  1. 数字在计算机中的编码方式：

    • 原码：符号位+绝对值的二进制表示
    • 反码：正数的反码与原码相同，负数的反码等于除符号位外其他位取反
    • 补码：正数的反码与原码相同，负数的反码等于除符号位外其他位取反+1

  2. 冯-诺依曼体系：控制器+计算器+存储器+输入设备+输出设备

计算机网络

  1. OSI模型：
    • 应用层
    • 表示层（在五层模型中合并到应用层）
    • 会话层（在五层模型中合并到应用层）
    • 传输层：TCP/UDP
    • 网络层：路由器
    • 数据链路层：交换机/网桥/网卡（半链路层半网络层）
    • 物理层：中继器/集线器/双绞线
  2. 路由器判断路由好坏的标准是路由权，它是根据各种路由信息计算得到的，权值越小的路由越好。

算法

  1. 傅里叶变换 快速傅里叶变换

  2. 主定理：

    假设一个方法F的时间复杂度是规模n的函数，且该函数由递归式 $T(n)=a(T(n/b))+f(n)$定义，则：

    • 当 $\exist \epsilon >0$,使得 $f(n)=O(n^{log_ba-\epsilon})$时，$T(n)=\Theta(n^{log_ba})$.
    • 当$f(n)=\Theta(n^{log_ba})$时，$T(n)=\Theta(n^{log_ba}log(n))$.
    • 当 $\exist \epsilon >0$,使得 $f(n)=\Omega(n^{log_ba+\epsilon})$时，且对常数 $c<1$和足够大的 $n$，有 $af(n/b)<=cf(n)$，则 $T(n)=\Theta(f(n))$.

  3. $T(n)=2T(n-1)+k(n>0, k>0), T(0)=c(c>0)$ -> $T(n)=O(2^n)$. 证明

  4. 分支限界法：对解空间树进行广度优先遍历，找到最优解 参考

  5. 基数排序：对于一个由若干n进制数构成的数组，可以这样排序，首先根据末位将所有数依次放入n个桶中（分配），然后将所有桶中的元素按顺序连接起来（收集）；再根据倒数第二位进行分配与收集…最后根据第一位进行分配与收集之后，就得到了一个有序的序列。假设共m个数，每个数最多包含d位，则基数排序的时间复杂度为O(d*(n + m))。

  6. 求1-n之间的所有数的因数：

    • 直接求解法：空间$O(1)$，时间$O(n\sqrt n)$
    • 倍数记录法：令i从1到n遍历，对每个i，令所有i的倍数都增加一个因子i。空间$O(n)$（假设平均每个数只有常数个因子），时间$O(nln(n))$（因欧拉公式 $\sum_{i=1}^n1/i=ln(n)+C$，其中 $C$ 为欧拉常数）

  7. 快速求幂：假设求 $a^k$ ，可以将指数 $k$ 写成二进制形式 $k=s_ms_{m-1}…s_0$ ，这样有 $a^k=a^{s_m…s_0}=a^{s_m2^m}*…*a^{s_02^0}$ 然后分别求$a^2,a^4,a^8,..,a^{2^m}$，最后连乘。相比于朴素算法的 $O(k)$ 复杂度，快速求法的复杂度为 $O(log_2k)$。

  8. 线性变换：求一个向量经过若干次相同线性变换所得的结果，可以对这个线性变换矩阵求快速幂，然后与原向量相乘。

机器学习

  1. 维度灾难：

    当空间的维数增加时，为了保持同样的采样密度，所需的采样点个数呈指数级增加，这样的现象称为维度灾难。比如，在一维单位区间上采样时，为了使最大采样距离不超过0.1，只需取10个采样点即可；而在10维单位超正方体里，为了使最大采样距离（在各个维度上）不超过0.1，需要有10^10个采样点。在机器学习中，当特征维度很大时，就会出现维度灾难，因为机器学习需要根据训练数据来估计真实的数据分布，而在高维特征空间，数据非常稀疏，任意两个数据（采样点）之间都可能距离很远，使得估计变得非常困难。

  2. 逻辑斯蒂回归中L范数正则化的作用：

    • 使一些权重趋近于0，从而达到特征筛选的目的，防止过拟合和维度灾难
    • 在梯度下降时，由于一些权重的值接近0，因而计算速度也增加了

编程题

  1. 对数组中每个元素，求在它后面且比它大的第一个元素 -> 单调栈
  2. n个相同小球放入m个相同盒子里（允许为空）的方法数 -> $P(n, m)=P(n-m, m)+P(n, m-1)$
  3. n个相同小球放入m个相同盒子里（不允许为空）的方法数 -> $Q(n, m)=P(n-m, m)$

C/C++

  1. 对于一个在栈上分配的数组a，a实际是数组本身，而不是一个指向该数组所占的内存空间的指针。但为了使用方便，C/C++被设计成可以将a当做指针来使用。比如：
char a[10];
sizeof(a); -> 10
scanf("%s", &a); -> 正确，&取数组a的地址
scanf("%s", a); -> 同样正确，a直接被当做指针（地址）来使用
  1. 声明n维数组时，必须指定后n-1维的长度。这是因为C/C++按照行优先的规则来存储数组元素，因此在根据元素下标来访问元素时，必须知道后n-1维的长度才能计算内存地址。
int a[2][5];
int b[][10];
int c[2][]; -> 错误
