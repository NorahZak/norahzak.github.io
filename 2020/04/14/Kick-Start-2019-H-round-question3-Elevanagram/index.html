Kick Start 2019 H round question3: Elevanagram

2020-04-14

#kickstart #programming

问题描述

给定若干1-9的数字，其中i的数目为A[i]，问如何排列这些数字，使得它们的alternative sum能够整除11？一列数的alternative sum定义为：a1-a2+a3-a4+...，比如81743的alternative sum为8-1+7-4+3=13。

解决方案

这个问题等价于将所有数字等分为两部分：正集合P与负集合N，其中P中的元素个数为 ceil(sum(A)/2)​，使得P中所有数的和与N中所有数的和之差能够整除11。

当所有A[i]都很小时，可以用动态规划来解决这个问题。dp[i][j][k]表示：当只考虑1-i的数字时，如果在P中放入j个数字，是否能使 (sum(P)-sum(N))%11=k .

  1. 初始化：i=0，此时只有dp[0][0][0]=True，其他所有dp[0][j][k]=False.
  2. 递推：为了求出dp[i+1](i>0)，首先将所有dp[i+1][j][k]初始化为False。对一切dp[i][j][k]=True，试着将A[i+1]个(i+1)分配到P和N中。如果将p个(i+1)分配到了A[i+1]中，那么就会有A[i+1]-p个(i+1)被分配到N中，此时P中就有j+p个元素，而模11的余数为 mod=(k+p*(i+1)-(A[i+1]-p)*(i+1))%11，因此dp[i+1][j+p][mod]=True。

动态规划的时间复杂度为O(9*sum(A)*11*max(A))，空间复杂度为O(11*sum(A))（如果采用轮换法）。

进一步地，当A[i]很大时（test set 2中A[i]<=10^9），动态规划的复杂度也会很大，此时该怎么做呢？我们需要借助于以下引理：当至少2个A[i]>=10时，解一定是存在的。

证明：假设1<i<j<10，且A[i]>=10，A[j]>=10，那么取10个i和10个j组成集合S，将剩下的所有元素随意均分为两半。接下来从S中选p个i和10-p个j放入P中，将S中剩下的10-p个i和p个j放入N中（0<=p<=10）。在p从0遍历到10的过程中，sum(P)-sum(N)每次都增加(j-i)*2，由于(j-i)*2与11是互素的，因此[sum(P)-sum(N)]%11遍历了0到10的所有值。

这样，当有两个A[i]>=10时，我们可以直接得出结果。而当只有1个A[i]>=10时（有可能很大，因此直接动态规划会内存爆炸），我们可以先对除了i之外的其余8个数进行动态规划（其余8个数的数目之和m不会大于72），最后对i单独处理。
