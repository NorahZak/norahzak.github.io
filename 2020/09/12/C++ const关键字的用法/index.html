C++ const关键字的用法

作用

在c++中，const关键字用来表示一个对象是不可变的。具体地，const关键字可以用来修饰基本数据类型、对象、类成员函数和指针。

const修饰基本数据类型

const修饰基本数据类型时，表示这是一个常量，不可修改。

const int a = 0;
a = 1;	// 编译错误

const修饰类的实例和类的函数

const修饰类的实例时，表示该实例是不可修改的，其中的数据成员也是不可修改的。对该实例，我们只能调用那些不可能修改其数据成员的成员函数，这些成员函数也应该用const关键字来修饰。当然，const成员函数是不能调用非const成员函数的（否则仍有修改数据成员的风险）。

class A{
    public:
        void f1() const{	# const函数
            cout << "f1 test" << endl;
        }
        void f2(){
            cout << "f2 test" << endl;
        }
};

int main(){
    const A b;
    b.f1();
    b.f2();	// 编译错误
}

const修饰指针

根据const的限制域不同，可分为以下两种情况：

  1. const *：此时不能通过该指针来修改其指向的变量，但该指针本身的值（一个内存地址）可变。

    int a = 1;
    const int *p = &a;	// p是const *型指针
    a = 2;							// p指向的变量是可变的
    cout << *p << endl;	// 输出：2
    int b = 3;
    p = &b;							// p的值可变（可以指向其他内存地址）
    cout << *p << endl;	// 输出：3
    *p = 4;							// 编译错误，不能通过p对其指向的变量进行修改

  2. * const：此时该指针本身的值（一个内存地址）不可变，但能通过该指针来修改其指向的变量。

    int a = 1;
    int* const p = &a;	// p是* const型指针
    int b = 2;
    p = &b;							// 编译错误，p的值不可变（不能指向其他内存地址）
    *p = 2;							// 可以通过p对p指向的变量进行修改
    cout << *p << endl;	// 输出：2
