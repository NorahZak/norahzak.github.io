1.压缩/解压缩

.tar/.tar.xz
解包：tar xvf FileName.tar [-C dst_dir]
打包：tar cvf FileName.tar DirName

.gz
解压1：gunzip FileName.gz
解压2：gzip -d FileName.gz
压缩：gzip FileName

.tar.gz/.tgz
解压：tar zxvf FileName.tar.gz
压缩：tar zcvf FileName.tar.gz DirName

.bz2
解压1：bzip2 -d FileName.bz2
解压2：bunzip2 FileName.bz2
压缩： bzip2 -z FileName

.tar.bz2
解压：tar jxvf FileName.tar.bz2
压缩：tar jcvf FileName.tar.bz2 DirName

.bz
解压1：bzip2 -d FileName.bz
解压2：bunzip2 FileName.bz
压缩：未知

.tar.bz
解压：tar jxvf FileName.tar.bz
压缩：未知

.Z
解压：uncompress FileName.Z
压缩：compress FileName

.zip
解压：unzip FileName.zip [-d DstDir]
压缩：zip FileName.zip DirName

.rar
解压：rar x FileName.rar
压缩：rar a FileName.rar DirName

.lha
解压：lha -e FileName.lha
压缩：lha -a FileName.lha FileName

.rpm
解包：rpm2cpio FileName.rpm | cpio -div

.deb
解包：ar p FileName.deb data.tar.gz | tar zxf -

.tar .tgz .tar.gz .tar.Z .tar.bz .tar.bz2 .zip .cpio .rpm .deb .slp .arj .rar .ace .lha .lzh .lzx .lzs .arc .sda .sfx .lnx .zoo .cab .kar .cpt .pit .sit .sea
解压：sEx x FileName.*
压缩：sEx a FileName.* FileName

sEx只是调用相关程序，本身并无压缩、解压功能，请注意！

gzip 命令
减少文件大小有两个明显的好处，一是可以减少存储空间，二是通过网络传输文件时，可以减少传输的时间。gzip 是在 Linux 系统中经常使用的一个对文件进行压缩和解压缩的命令，既方便又好用。

语法：gzip [选项] 压缩（解压缩）的文件名该命令的各选项含义如下：

-c 将输出写到标准输出上，并保留原有文件。-d 将压缩文件解压。-l 对每个压缩文件，显示下列字段： 压缩文件的大小；未压缩文件的大小；压缩比；未压缩文件的名字-r 递归式地查找指定目录并压缩其中的所有文件或者是解压缩。-t 测试，检查压缩文件是否完整。-v 对每一个压缩和解压的文件，显示文件名和压缩比。-num 用指定的数字 num 调整压缩的速度，-1 或 –fast 表示最快压缩方法（低压缩比），-9 或–best表示最慢压缩方法（高压缩比）。系统缺省值为 6。

gzip *% 把当前目录下的每个文件压缩成 .gz 文件。gzip -dv *% 把当前目录下每个压缩的文件解压，并列出详细的信息。gzip -l *% 详细显示例1中每个压缩的文件的信息，并不解压。gzip usr.tar% 压缩 tar 备份文件 usr.tar，此时压缩文件的扩展名为.tar.gz。

.iso文件解压：Windows下可以用7z或者WinRAR，Linux上这两个都不行（7z在Linux上只有7za版本，而7za不支持.iso文件解压；rar的Linux版本不支持.iso文件），可以用xorriso，参考链接：https://superuser.com/questions/180744/how-do-i-extract-an-iso-on-linux-without-root-access

2.scp文件传输

格式：scp [-r] [-P] src_dir des_dir

-r：传输文件夹

-P：指定传输端口

src_dir：待传输文件的路径

des_dir：传输目的地

远程主机路径的格式：username@ip:/…

scp -r -P 8003 Downloads/benchmark_RELEASE wjzhang@219.223.187.220:

3.Linux服务器上使用JupyterNotebook

安装JupyterNotebook

pip install jupyter

建立端口映射

JupyterNotebook的运行原理：首先在本地创建一个HTTP Server，它占用一个端口（默认为8888），然后我们可以通过HTTP协议访问该端口（也就是用浏览器访问http://localhost:8888）来访问notebook。但是，在服务器上运行JupyterNotebook时，由于服务器上没有浏览器，所以我们想要用本地的浏览器访问服务器上的Jupyter Notebook，这时应该怎么做呢？

（1）建立SSH端口映射，将对本地端口的访问重定向到服务器上的Jupyter Notebook Server：

ssh -L localhost:port1:localhost:port1 username@ip -p port0

  • **-L**：将远程端口映射到本地
  • **port1**：本地端口号
  • **port2**：远程的Jupyter Notebook Server占用的端口号
  • **port0**：通过SSH连接到远程服务器使用的端口号
ssh -L localhost:${local-port}:localhost:8888 user@${remote-ip} -p ${remote-port}

（2）在服务器上运行jupyter-notebook：

jupyter-notebook

（3）在本地的浏览器中，使用localhost:${local-port}来访问服务器上的jupyter notebook：

http://localhost:${local-port}

（4）如果是第一次使用，会要求输入密码或者口令。在出现的窗口中，输入token，token的值在连接到服务器的终端上可以看到：

To access the notebook, open this file in a browser:

file:///home/user/.local/share/jupyter/runtime/nbserver-92727-open.html

  Or copy and paste one of these URLs:

    http://localhost:8888/?token=190c3188d22cafb524742b243558723257b8d83509fbbb5b

   or http://127.0.0.1:8888/?token=190c3188d22cafb524742b243558723257b8d83509fbbb5b

（5）使用Jupyter Notebook。

6.非root用户安装软件包

参考链接：https://tlanyan.me/work-with-linux-without-root-permission/

在linux系统下安装软件包，有两种方式：

（1）直接用软件管理工具来下载（比如apt-get, yum等等）

（2）下载源码后自行编译安装

无论哪种方式，默认的安装目录通常是/usr/bin或/use/local/bin，而对这两个目录，非root用户都是只有读和执行权限，没有写权限的。也就是说，非root用户可以执行这两个目录下的程序，却不能将新程序安装在这两个目录下，只能将程序安装到自己有写权限的目录下面。因此，非root用户在安装新软件时，必须修改安装路径，而这只有（2）源码安装方式能够做到。

操作步骤如下：

（1）下载源码：下载源代码到用户目录

（2）编译：解压源码包并进入源码目录，然后使用make命令编译源码，得到可执行程序

（3）配置安装路径：在源码目录下一般有configure.sh文件，它可用来修改配置。使用./configure --help命令来查看如何修改配置；通常使用./configure --prefix=/home/me/dir来将安装路径由默认的/usr/bin 或/use/local/bin修改为你自己的安装路径（必须是一个你有写权限的目录，比如可以是你用户目录下的子目录，如home/me/dir）。有时配置过程中会提示你缺乏某些依赖包。如果没有configure.sh文件，可能要在install.sh中修改安装路径

（4）安装：通常源码目录中会有install.sh，直接执行该脚本即可安装程序（其实就是将（2）中生成的可执行程序拷贝到安装目录下面）

（5）修改环境变量：编辑你所用shell的配置文件（如果你用的是zsh，那么该配置文件的路径为~/.zshrc），在其中加入export "/home/me/dir/bin:$PATH"这一条语句，其中/home/me/dir是你在第（3）步的安装路径，然后使用source ~/.zshrc命令使修改立即生效

（6）使用：接下来就可以愉快地使用软件了

7.Linux服务器上使用Matlab

1.安装

（1）下载并解压：

首先从清华info下载Matlab的映像文件（.iso文件），然后在Linux系统下用xorriso将其解压，这里解压路径为home/me/matlab

xorriso -osirrox on -indev R2019a_Linux.iso -extract / matlab

（2）安装：

home/me/matlab目录下有install文件用于安装。由于服务器没有图形化界面，因此需要采用静默安装方式，这样就需要我们在home/me/matlab目录下的安装配置文件install_input.txt中手动修改配置，然后再安装。除此之外，激活文件activate.ini也需要修改。

install_input.txt的修改：

destinationFolder= /home/me/my-matlab //安装目录
fileInstallationKey=38699-60149-36808-21840-05491 //你的序列号
agreeToLicense=yes //同意协议
outputFile=/tmp/mathwork_install.log //安装日志
mode=silent //开启无人值守安装
activationPropertiesFile=/home/me/matlab/activate.ini //激活文件
licensePath=/home/me/matlab/network.lic //你的license文件

其中license文件和序列号都从清华info的matlab安装手册中获取。

activate.ini的修改：

isSilent=true //开启silent模式
activateCommand=activateOffline //设置激活方式, 离线激活 无需联网
licenseFile=/home/me/matlab/network.lic //license文件位置

修改完配置之后即可开始安装，注意：安装应在home/me/matlab目录之外进行；inputFile必须用绝对路径；安装模式设置为静默安装。

./matlab/install -inputFile /home/me/matlab/installer_input.txt -mode silent

（3）配置环境变量：

如果你没有使用默认的安装路径，则需要配置环境变量，将安装路径下的bin目录加入到环境变量中。

（4）使用：

在命令行中输入matlab，即可打开matlab命令行。

2.使用

Matlab有两种执行程序的方式：

  • 命令行下交互式输入命令及执行
  • 执行脚本（.m文件）

为了执行.m文件，需要进入.m文件的目录，并输入以下命令：

matlab -nodesktop -nosplash -r xxx
  • -nodesktop : 无图形界面（由于服务器没有图形界面，因此在服务器上执行.m文件必须加入此参数）
  • -nosplash : 不显式启动画面

如果要在后台执行.m文件，需要输入以下命令：

nohup matlab -nodesktop -nosplash < xxx.m &

注意，不要用nohup matlab -nodesktop -nosplash xxx & 这样的语句，否则当.m文件中有文件读写操作时，会出现Bad file discripter错误。（原因不明）

command > file: 将输出重定向至文件file

command >> file: 将输出以追加的方式重定向至文件file

command < file: 将输入重定向至文件file

9.查看进程详细信息

当一个进程被创建后，linux会在/proc目录下创建一个与该进程的进程号同名的文件夹，其中存储了该进程的详细信息。

ll /proc/pid即可查看该目录下所有文件的详细信息。

cat /proc/pid/cmdline即可查看创建该进程时在命令行中输入的指令。

10.SSH免密登录

假设有A与B两台机器，现想在A上用SSH免密码登录到B，步骤如下：

  1. 生成A的公钥：使用ssh-keygen命令生成SSH公钥/私钥对，生成的公钥和私钥会保存为.ssh/id_rsa.pub和.ssh/id_rsa
  2. 将A的公钥加入B的已授权公钥中：B的已授权公钥存储在.ssh/authorized_keys文件中，将A的公钥.ssh/id_rsa.pub复制到B的.ssh/authorized_keys中，并将文件权限设为600.
  3. 接下来即可从A免密登录到B
